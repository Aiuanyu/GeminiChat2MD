請講客話，請用 ROC 教育部+客委會个正式用字：𠊎有一隻安到 HakSpring 个專案， 詞目文字顯示在 table cell 裡肚， 讀音用 ruby 文字標註， 有兜詞目較長， ruby 元素在 Chrome 做得換行 wrap，毋會超出 td 邊界 其他瀏覽器方面，原旦用 adjustRuby... 相關函式動態調整 ruby 文字大細 毋過在 WebKit 瀏覽器又過無效

敢有可能在其他瀏覽器改用毋係 ruby 毋過還係上下對照讀音个排版？inline-table？

𠊎來同你討論這個問題。你个情形係 WebKit 瀏覽器對 ruby 元素个支援無恁好，特別係在表格裡肚个換行處理。

建議解決方案

1. 用 CSS Grid 來模擬 ruby 排版

html
<td>
  <div class="hakka-ruby">
    <div class="hakka-text">客家話</div>
    <div class="hakka-phonetic">hag5 ga1 fa4</div>
  </div>
</td>
css
.hakka-ruby {
  display: inline-grid;
  grid-template-rows: auto auto;
  justify-items: center;
  line-height: 1.2;
  vertical-align: baseline;
}

.hakka-text {
  grid-row: 2;
  font-size: 1em;
}

.hakka-phonetic {
  grid-row: 1;
  font-size: 0.5em;
  color: #666;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
  max-width: 100%;
}

2. 用 inline-block 配 data 屬性

html
<td>
  <span class="hakka-word" data-phonetic="hag5 ga1 fa4">客家話</span>
</td>
css
.hakka-word {
  display: inline-block;
  position: relative;
  padding-top: 1em;
}

.hakka-word::before {
  content: attr(data-phonetic);
  position: absolute;
  top: 0;
  left: 50%;
  transform: translateX(-50%);
  font-size: 0.5em;
  color: #666;
  white-space: nowrap;
}

3. 瀏覽器偵測個折衷方案

javascript
function setupHakkaDisplay() {
  const isWebKit = /webkit/i.test(navigator.userAgent) && 
                   !/chrome/i.test(navigator.userAgent);
  
  if (isWebKit) {
    // WebKit 用替代方案
    document.querySelectorAll('ruby').forEach(ruby => {
      const rb = ruby.querySelector('rb') || ruby.childNodes[0];
      const rt = ruby.querySelector('rt');
      
      if (rb && rt) {
        const wrapper = document.createElement('span');
        wrapper.className = 'hakka-ruby-alt';
        wrapper.innerHTML = `
          <span class="hakka-base">${rb.textContent}</span>
          <span class="hakka-annotation">${rt.textContent}</span>
        `;
        ruby.parentNode.replaceChild(wrapper, ruby);
      }
    });
  }
  // Chrome 摎其他瀏覽器用原本个 ruby
}
css
.hakka-ruby-alt {
  display: inline-flex;
  flex-direction: column-reverse;
  align-items: center;
  vertical-align: bottom;
  line-height: 1;
}

.hakka-annotation {
  font-size: 0.5em;
  color: #666;
  line-height: 1.2;
}

.hakka-base {
  line-height: 1.5;
}

4. 表格裡肚个響應式處理

css
td {
  max-width: 200px; /* 設定最大闊度 */
  word-wrap: break-word;
  overflow-wrap: break-word;
}

/* 針對較長个詞目 */
@media (max-width: 768px) {
  .hakka-phonetic {
    font-size: 0.4em;
  }
  
  td {
    font-size: 0.9em;
  }
}

這兜方案个優點：

  • Grid 方案：對齊效果最好，支援度高
  • Data 屬性方案：語義清楚，維護簡單
  • 瀏覽器偵測：做得保留原本 ruby 个語義結構
  • 響應式處理：在毋同螢幕大細都做得正常顯示

你做得照你个需求選擇最適合个方案，或者結合使用。愛有麼个其他問題，歡迎續問！
